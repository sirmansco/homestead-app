## 1. Spec

### Group A ‚Äî Safe-Area Scroll (3 bugs)

**BUG-2:** Submit button on Lantern compose form sits below the safe-area-inset on devices with home indicator (iPhone 14+). Static `paddingBottom: 100px` does not stack with the device's safe area, so the submit button is partially obscured.

**BUG-3:** After tapping a text field on the Lantern form, the iOS keyboard pushes the page up and the masthead scrolls above the safe area. Drag-down gesture restores it but this is broken UX. Root cause: the scroll container has `overflowY: 'auto'` but the parent flex column doesn't use `height: 100%` or `minHeight: 0` correctly with keyboard resize.

**UX-2:** Same static 100px bottom padding pattern exists in `ScreenPerch`, `ScreenWhistles`, `ScreenCircle`, and `ScreenPost` scroll containers ‚Äî content is clipped on large-safe-area devices. Consistent fix across all five screens.

### Group C ‚Äî UX Copy / Polish (4 items)

**UX-1:** Empty state copy on the Perch is functional but cold. Should invite action more warmly.

**UX-3:** "ALL" tab in Whistles shows shifts from all households but label reads generically. When a watcher is in multiple households the label should contextualize.

**UX-4:** Covies card in ScreenCircle shows a KEEPERS column but no WATCHERS column. Watchers in the circle have no visibility in the Covies view.

**UX-6:** "‚Äπ Back" button on Lantern compose is present in code but visually indistinct ‚Äî same weight/color as body text, easy to miss.

### Out of scope for this plan

- BUG-W2 (avatar photo upload failure): needs API-layer investigation, separate fix
- Multi-household caregiver switcher UI: open design question, not audited

## 2. Conventions

- **Inline styles only** ‚Äî no Tailwind, no CSS modules. All layout via JS style objects.
- **Design tokens** from `./tokens`: `G`, `RED`, `RED_DARK`, `BELL_BG`. Never hardcode hex values that have a token.
- **Safe-area pattern already established** ‚Äî search for `calc(32px + env(safe-area-inset-bottom, 0px))` in `ScreenCircle.tsx` (bottom sheets use it). Scroll containers should use same `env()` form.
- **`getCopy()`** for all user-facing strings ‚Äî never hardcode display text inline.
- **`useAppData()`** for shared state. No prop-drilling for context values available there.
- **No silent catch** ‚Äî all async catch blocks must set error state, not swallow.
- **No stacking rule**: if fix touches 3+ unrelated files in one commit, flag it first.

## 6. Fragile Areas

- **Keyboard + scroll on iOS Safari**: `visualViewport` resize behavior differs across iOS versions. If adding `dvh`, test on iOS 15 (no `dvh` support) and fall back to `100vh` with safe-area compensation. This is the riskiest change in Group A.
- **ScreenCircle Covies WATCHERS column**: `village` data is available via context but the shape needs verification ‚Äî confirm `ActiveBellData` or the village member type includes `role` field before rendering the filter. See `app/context/AppDataContext.tsx` for the type definition.
- **Whistle stream** (`enableWhistleStream`): `ScreenPerch` uses SSE; any layout change to the scroll container should not disrupt the stream subscription. The subscription is in `useEffect` on mount, not tied to layout.
- **Tab bar z-index**: Tab bar sits above the scroll container. If changing padding values, confirm the tab bar's `position: fixed` and `bottom: env(safe-area-inset-bottom, 0px)` is not double-counting with scroll container padding.
